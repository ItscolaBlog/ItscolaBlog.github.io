一. 这就是变量

1. 如何理解变量

想象两个快递盒子，分别装着你买的键盘和鼠标。如果有人问你这两个分别是什么？ 你可以回答 “这个是鼠标，这个是键盘” 。

这时，我们把 装有鼠标的盒子 和 装有键盘的盒子，分别看作一个整体。 如果我们把这两个盒子里的物品互换，鼠标装入原有键盘的盒子，键盘装入原有鼠标的盒子。 那这时，我可以指着原本被指代鼠标的盒子说 “现在这是键盘了”。

是盒子变了吗？并不是，是盒子里的物品变了。

我们可以把 快递盒子看作变量，而 变量表示的值看作快递盒子里装的物品。

比如 ： 变量 box 的值为 10 ，可以想象成 一个被称为 box 的盒子，它里面装的物品为 “10”。 那我们只要一看到 box 就可以将它可以指代为 “10”了。

那如果我把 box 的值变为了 11，那是不是就类似于 我把盒子 “box” 里面的物品换成 “11”了？ 是的，变量就那么简单。

2. 变量的概念

那么，变量是用一个 “盒子”来表示“盒子所装的物品”。并且，“盒子”所表示的里面的“物品”是可以变成其它“物品”的。

我们把“盒子”的名字，称为 变量名。
我们把“盒子”所代表的“物品”，称为 变量值。

那么可以说：变量是用一个变量名，来指代可能会发生改变的变量值。

学习是用旧的概念来理解新的概念的过程，我希望你看完这篇文章，也能用自己的话总结出什么是变量。

二. 变量与 Java

1. 变量与编程语言

目前为止，绝大部分的编程语言，都有变量这一概念，并且含义都类似。 理解了变量的概念在其它编程语言也通常适用。

虽然变量是基础中的基础，但离开了变量，这些编程语言就失去了灵魂。（也许以后会出现更先进的方式，但目前是这样。）

2. 变量在 Java 中的表示

int a = 12;
我们先看 “int”的后面部分，
这里的 “=” 并不是数学上含义的等号，而是表示 “赋值” 。 这就像你把 12 这个物品，装进 a 这个盒子中，然后我们会用盒子 a 来指代 物品 12。

也就是 “把变量值 12 赋予给名为 a 的变量，让变量 a 来表示 变量值 12。”
那么 a 变量就指代了它的值 12。

再来看前面的 “int” 它表示了变量 a 可以被赋予值的类型。
就好像不同的盒子能装的物品类型不一样，你不可能将放鼠标的快递盒子装一头大象，也不能将放鼠标的盒子装一个蛋糕。

int 表示整数类型，表示 a 这个变量只能 = 为整数型。 int 为 “Integer” 这个 单词的缩写，也就是整数。（后门还会讲 long 类型）

11 是整数 没问题。 int a = 11;
100 是整数 没问题。 int a = 100;
1001是整数 没问题。 int a = 1001;

“apple” 不是整数，它是“一行字” 不行 不能有 int a = “apple”

3. 变量在 Java 中的操作

代码是从上往下一行一行走的，第一个小任务，猜测下方四行代码的作用，就当热身吧。 也不用追求完美，热热身就好。

int a = 10;
int b = 11;
a = 12;
b = a;

这就是这节课你的学习目标，看懂这四行代码。不需要你打开电脑跟着操作，我会在每章最后一节准备一个实战篇，非必要情况下，实战篇之前，你就理解概念就好。

新手都特别喜欢学点内容和其它人炫耀？ 也理解你们，你可以仿照这四行代码写出一些操作，那也算你自己写的了。 但要知道，这是第一两节课程，所以内容是基础中的基础，向别人炫耀失败也很正常。 或是暗中努力让别人惊叹吧。

先看第一行代码 int a = 10; 将 10 赋值给 int 型（整数型）变量 a。

第二行代码 int b = 11;
将 11 赋值给 int 型（整数型）变量 b。

第三行代码 a = 12;
将 12 赋值给 int 型（整数型）变量 a。
这时候 变量 a 表示的就不是 10 了，而被换成了12。 就好像把盒子里的键盘还成了鼠标。

这里为什么没有 int ？ 因为我们在第一行已经声明 a 为 int 类型了。 变量的声明一次就好了。
我们还可以先声明再赋值，但不能先赋值再声明。 第一行注意声明同时赋值。

// 先声明再赋值；
int a;
a = 10;

// 声明同时赋值
int a = 10;

再来看最后一行代码 b = a;
意思是 b 和 a 相等吗？ 并不是！ 我们已经说过 ，编程中一个 = 代表赋值 ，不是代表等于。

它的意思是将变量 a 赋值给 b。 也就是把变量 a 代表的值 12 赋字给变量 b， 这样 b 代表的值就从原来的值 11 变为 12 了。

和比喻成盒子不一样的是，a 盒子里的物品 12 放到 盒子 b 里。此时 b盒子里的物品也是 12，但是 a 盒子里的物品还是 12，没有消失。 这就是 = 在赋 整数值时候的特性，这里的赋值就像复制了 a 盒子里的物品，然后放到 b 盒子里，然后 b 盒子里的物品就被覆盖了，变成了 a 的物品 12。因为 java 中，变量只能表示一个值，不能表示多个。 不能一个盒子里有两个物品。

所以这些代码最后走下来，变量 a 和变量 b 都代表值 12。

三. 这就是 Java 中的一个概念

变量：变量是用一个变量名，来指代可能会发生改变的变量值。

变量名：变量的名字。

变量值：变量所代表的值。

= ： 赋值，将后面的值赋给前面的变量。

代码格式： 类型 变量名 = 变量名或值;
如 int a = b; int a = 10;
